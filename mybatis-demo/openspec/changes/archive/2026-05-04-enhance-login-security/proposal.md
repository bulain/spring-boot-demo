## 为什么

当前的用户名密码登录缺少必要的安全防护机制。存在以下安全风险：
1. 攻击者可以通过暴力破解尝试密码
2. 同一 IP 可以无限次尝试登录
3. 用户可以设置过于简单的密码
4. 账户被暴力破解时没有锁定保护

为了加强系统安全性，需要实现完整的登录安全防护机制。

## 变更内容

- 新增密码错误次数统计和账户自动锁定功能
- 新增登录失败 IP 限流机制
- 新增用户密码复杂度校验（创建用户和修改密码时）
- 新增防暴力破解机制（IP + 账户双重维度保护）
- 锁定的账户需要管理员解锁或自动解锁（超时后）
- **新增登录图片验证码**：当同一 IP 或同一账户登录失败达到指定次数后，需要输入图片验证码才能继续登录

## 能力

### 修改的能力
- `user-login`: 用户名密码登录现在包含密码错误次数限制、IP 限流、账户锁定检查、图片验证码验证
- `image-captcha`: 图片验证码能力扩展，支持登录场景

### 新增的能力
- `password-policy`: 密码复杂度校验和安全策略
- `login-rate-limit`: 登录失败限流和账户锁定机制
- `login-captcha`: 登录验证码触发和验证机制

## 影响

- `src/main/java/com/bulain/mybatis/sys/controller/SysAuthController.java`: 添加登录安全检查逻辑、图片验证码验证逻辑
- `src/main/java/com/bulain/mybatis/sys/service/SysUserService.java`: 添加密码复杂度校验、账户锁定/解锁方法
- 新增登录安全服务类：用于统计登录失败次数、IP 限流、账户锁定管理、验证码触发规则
- 使用 Redis 存储登录失败计数、锁定状态、验证码触发标记
- 登录接口返回值扩展：当需要验证码时返回相应的提示和 captchaId
- 可能影响：用户创建、重置密码、登录等相关接口
