## 为什么

`/api/sys/auth/phone-login` 接口目前仅通过手机号查找用户，但不验证提供的验证码是否有效。这是一个安全漏洞——只要手机号存在，任何验证码都能访问账户。

## 变更内容

- 新增发送短信验证码接口（需先验证图片验证码）
- 在手机登录接口中添加短信验证码校验逻辑
- 集成验证码服务（使用 Redis 存储）来生成和验证各类验证码

## 能力

### 新增的能力
- `send-verification-code`: 验证图片验证码通过后，向指定手机号发送短信验证码
- `image-captcha`: 生成和验证图片验证码

### 修改的能力
- `phone-login`: 手机号+验证码登录现在可以正确验证验证码

## 影响

- `src/main/java/com/bulain/mybatis/sys/controller/SysAuthController.java`: 新增获取图片验证码接口，新增发送短信验证码接口，添加验证码校验逻辑
- 新增验证码服务类：用于生成、存储和验证图片验证码及短信验证码
- 使用 Redis 存储验证码和计数（项目已集成 Redisson）
- 获取图片验证码：生成随机验证码文本，生成图片，存储到 Redis（设置过期时间），返回图片
- 发送短信验证码时：先验证图片验证码，通过后生成随机短信验证码，存储到 Redis（设置过期时间），调用短信服务发送
- 登录验证逻辑：从 Redis 读取该手机号对应的短信验证码进行比对
- 次数限制（使用 Redis 计数）：
  - 图片验证码：同一 IP 1 分钟内最多获取 10 次
  - 短信验证码：同一手机号 1 小时内最多发送 5 次，1 天内最多发送 10 次
  - 超过次数限制时返回相应的错误提示
