## ADDED Requirements

### Requirement: 基于注解的权限校验
系统必须提供基于注解的权限校验功能，用于控制器方法。

#### Scenario: 用户拥有所需权限
- **WHEN** 拥有所需权限的用户访问受保护的端点
- **THEN** 系统允许请求继续执行

#### Scenario: 用户缺少所需权限
- **WHEN** 没有所需权限的用户访问受保护的端点
- **THEN** 系统返回 403 Forbidden 错误

#### Scenario: 用户拥有所需角色
- **WHEN** 拥有所需角色的用户访问受角色保护的端点
- **THEN** 系统允许请求继续执行

### Requirement: 编程式权限校验
系统必须提供用于编程式权限校验的API。

#### Scenario: 编程式检查用户是否拥有权限
- **WHEN** 应用调用权限检查API，传入用户ID和权限编码
- **THEN** 系统返回true如果用户拥有该权限，否则返回false

#### Scenario: 编程式检查用户是否拥有角色
- **WHEN** 应用调用角色检查API，传入用户ID和角色编码
- **THEN** 系统返回true如果用户拥有该角色，否则返回false

### Requirement: 禁用用户的权限校验
系统必须拒绝所有禁用用户的权限请求。

#### Scenario: 禁用用户尝试访问资源
- **WHEN** 被禁用的用户尝试访问任何受保护资源
- **THEN** 系统拒绝访问，无论其分配的角色/权限如何
