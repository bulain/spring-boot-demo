# 导入模板下载 - 技术调研文档

**创建日期**: 2026-05-05
**功能**: 导入模板下载

## 技术决策

### 1. Excel生成库选择

**决策**: 继续使用 EasyExcel（项目已有依赖）

**Rationale**:
- 项目在用户导入和角色导入功能中已使用 EasyExcel，技术栈统一
- EasyExcel 支持流式读写，内存占用低，适合生成模板文件
- 支持多Sheet生成，便于实现"数据填写页 + 说明页"的双Sheet结构
- API简洁，易于维护

**备选方案考虑**:
- Apache POI - 内存占用高，API复杂，不推荐
- OpenCSV - 纯CSV格式，无法支持多Sheet、样式等需求

### 2. 模板生成方式

**决策**: 动态生成模板（无需静态文件）

**Rationale**:
- 模板文件简单（仅包含表头和一行示例），动态生成更灵活
- 避免维护静态模板文件，减少文件路径问题
- 便于后续字段变更时统一修改代码
- 支持生成说明页Sheet，动态填写字段说明

**备选方案考虑**:
- 静态模板文件 - 需要维护文件，变更不够灵活

### 3. 中文文件名处理

**决策**: 使用 URLEncoder 编码 + Content-Disposition 响应头

**Rationale**:
- 符合 HTTP 标准规范，兼容性好
- 项目已有类似实现（用户导出、角色导出）可参考
- 支持主流浏览器（Chrome、Firefox、Safari、Edge）

### 4. 模板Sheet结构

**决策**: 双Sheet结构（填写说明 + 数据模板）

**Rationale**:
- 符合之前澄清的需求：包含独立的说明页Sheet
- 说明页详细解释各字段格式要求、数据规则、注意事项
- 数据页仅包含表头和示例数据，保持简洁
- 用户下载后先阅读说明页再填写，减少导入错误

### 5. 权限控制

**决策**: 使用现有 Spring Security 机制，验证管理员权限

**Rationale**:
- 项目已有完善的权限控制体系
- 与其他管理接口保持一致的权限检查逻辑
- 无需新增复杂的限流机制，模板下载频率低
